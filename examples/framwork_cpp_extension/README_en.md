# Adaptation Development and Usage

## Overview

This example project demonstrates how to register a custom operator for both eager mode and `torch.compile` mode using native PyTorch `cpp_extension` features. For details about eager mode and `torch.compile` mode, see [Link](https://docs.pytorch.ac.cn/docs/stable/generated/torch.compile.html). 

## Directory Structure

```text
├── build_and_run.sh                // Script for compiling and installing the custom operator wheel package and executing the test case
├── csrc                            // Directory of the C++ code at the operator adaptation layer
│   ├── add_custom.cpp              // Adaptation code and forward-backward binding of the custom operator
│   ├── function.h                  // Header file of the forward and backward APIs
│   ├── pytorch_npu_helper.hpp      // Framework for calling and delivering custom operators.
│   └── registration.cpp            // Custom operator Aten IR registration file
├── custom_ops                      // Python code of the custom operator package
│   ├── add_custom.py               // Python API for the custom operator
│   └── __init__.py                 // Python initialization file
├── setup.py                        // Compilation file of the wheel package
└── test                            // Directory for test cases
    ├── test_add_custom_graph.py    // Script for executing test cases in torch.compile mode
    └── test_add_custom.py          // Script for executing operator test cases in eager mode
```

## Key Step Analysis of the `build_and_run.sh` Script

  - Compile the adaptation layer code and generate the wheel package:

    ```bash
    python3 setup.py build bdist_wheel
    ```

  - Install the compiled wheel package:

    ```bash
    cd ${BASE_DIR}
    pip3 install dist/*.whl
    ```

## Key Steps for Integrating Custom Operators into a Graph

  You can view the corresponding registration implementation in `test_add_custom_graph.py`.

  - Fill in the parameters for `torchair.ge.custom_op` based on the `REG_OP` prototype generated by the Ascend C project.

    The `REG_OP` prototype for `AddCustom` is defined as follows:

    ```cpp
    REG_OP(AddCustom)
        .INPUT(x, ge::TensorType::ALL())
        .INPUT(y, ge::TensorType::ALL())
        .OUTPUT(z, ge::TensorType::ALL())
        .OP_END_FACTORY_REG(AddCustom);
    ```

  - Register the custom operator converter:

    ```python
    from torchair import register_fx_node_ge_converter
    from torchair.ge import Tensor
    @register_fx_node_ge_converter(torch.ops.myops.add_custom.default)
    def convert_npu_add_custom(x: Tensor, y: Tensor, z: Tensor = None, meta_outputs: Any = None):
        return torchair.ge.custom_op(
            "AddCustom",
            inputs={"x": x, "y": y,},
            outputs=['z']
        )
    ```

## Running the Example Operator

This script is verified under PyTorch 2.1 and Python 3.9.

### 1. Building the Operator Project

Before running this example project, complete preparations by referring to [Building the Operator Project](https://gitee.com/ascend/samples/blob/master/operator/ascendc/0_introduction/1_add_frameworklaunch/README.md#operatorcompile).

### 2. Running the Example Project Using PyTorch

  - Go to the example project directory.

    Download the sample source code through the command line (using the `master` branch as an example):

    ```bash
    cd ${git_clone_path}/samples/operator/ascendc/0_introduction/1_add_frameworklaunch/CppExtensionInvocation
    ```

  - Sample Execution

    The execution workflow automatically generates test data, runs the PyTorch test cases, and validates the outputs. For details, see the build_and_run.sh script.
    
    ```bash
    bash build_and_run.sh
    ```

#### Additional Execution Instructions

  - After the environment setup is complete, you can run the test cases for `eager` mode and `torch.compile` mode separately.

    - Execute the custom operator test script in PyTorch eager mode:

      ```bash
      python3 test_add_custom.py
      ```

    - Execute the custom operator test script in PyTorch `torch.compile` mode:

      ```bash
      python3 test_add_custom_graph.py
      ```

### Note

    For more information about adapting operators for PyTorch, see the "Operator Adaptation Development Guide" in the [LINK](https://gitee.com/ascend/op-plugin/wikis).
