# OpPlugin

## Overview

This project develops the Ascend Extension for PyTorch (`torch_npu`) operator plugin (OpPlugin), enabling developers using the PyTorch framework to conveniently access NPU operator libraries.
The compilation and use of OpPlugin depend on Ascend Extension for PyTorch. Therefore, before compiling OpPlugin, you must first install Ascend Extension for PyTorch. For installation and usage instructions, see the Ascend Community documentation: [Ascend Extension for PyTorch](https://gitcode.com/Ascend/pytorch/blob/master/README.md?_fb=tree).

```ColdFusion
└─test                             # Test directory
```

## Installing OpPlugin

OpPlugin can be installed by building it from source. For details, see [Installing OpPlugin](docs/en/install.md).

## Quick Start

A complete development guide is provided for calling Ascend C custom operators in PyTorch through OpPlugin. It covers the entire workflow, including environment setup, operator registration, adaptation implementation, and test verification. For details, see [Example](https://gitcode.com/Ascend/pytorch/blob/v2.7.1/docs/zh/framework_feature_guide_pytorch/opplugin_operator_adaptation.md).

## API Reference

Provides function descriptions, prototypes, parameter descriptions, and call examples for custom Ascend Extension for PyTorch APIs based on PyTorch versions such as 2.10.0, 2.9.0, 2.8.0, and 2.7.1. For details, see [Custom API Reference](https://gitcode.com/Ascend/op-plugin/blob/master/docs/zh/custom_APIs/menu_Pytorch_API.md).

## Lifecycle

The `op-plugin` repository depends on `torch_npu` and follows its lifecycle. For details, see the [PyTorch Version Maintenance Strategies](https://gitcode.com/ascend/pytorch/blob/master/README.zh.md#pytorch%E7%89%88%E6%9C%AC%E7%BB%B4%E6%8A%A4%E7%AD%96%E7%95%A5) in the `torch_npu` repository.
